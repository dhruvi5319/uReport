**TicketRepository.java — add three FTS native queries:**

The `search_vector` tsvector column, GIN index `idx_tickets_search_vector`, and BEFORE INSERT/UPDATE trigger already exist (Phase 1 Flyway V2). Do NOT recreate them.

Add these three native @Query methods to the existing TicketRepository interface (which extends JpaSpecificationExecutor):

```java
// FTS without additional filters
@Query(value = """
    SELECT t.*,
           ts_headline('english', t.description, plainto_tsquery('english', :q),
               'MaxWords=30, MinWords=10, StartSel=<mark>, StopSel=</mark>, HighlightAll=false')
               AS search_snippet,
           ts_rank_cd(t.search_vector, plainto_tsquery('english', :q)) AS rank
    FROM tickets t
    WHERE t.search_vector @@ plainto_tsquery('english', :q)
    ORDER BY rank DESC, t.entered_date DESC
    LIMIT :pageSize OFFSET :offset
    """, nativeQuery = true)
List<Object[]> searchTickets(@Param("q") String q,
                             @Param("pageSize") int pageSize,
                             @Param("offset") int offset);

// FTS combined with status and categoryId filters (pass null to ignore each filter)
@Query(value = """
    SELECT t.*,
           ts_headline('english', t.description, plainto_tsquery('english', :q),
               'MaxWords=30, MinWords=10, StartSel=<mark>, StopSel=</mark>, HighlightAll=false')
               AS search_snippet,
           ts_rank_cd(t.search_vector, plainto_tsquery('english', :q)) AS rank
    FROM tickets t
    WHERE t.search_vector @@ plainto_tsquery('english', :q)
      AND (:status IS NULL OR t.status = :status)
      AND (:categoryId IS NULL OR t.category_id = :categoryId)
    ORDER BY rank DESC, t.entered_date DESC
    LIMIT :pageSize OFFSET :offset
    """, nativeQuery = true)
List<Object[]> searchTicketsWithFilters(@Param("q") String q,
                                        @Param("status") String status,
                                        @Param("categoryId") Long categoryId,
                                        @Param("pageSize") int pageSize,
                                        @Param("offset") int offset);

// Count for pagination when FTS active
@Query(value = """
    SELECT COUNT(*) FROM tickets t
    WHERE t.search_vector @@ plainto_tsquery('english', :q)
      AND (:status IS NULL OR t.status = :status)
      AND (:categoryId IS NULL OR t.category_id = :categoryId)
    """, nativeQuery = true)
long countSearchTickets(@Param("q") String q,
                        @Param("status") String status,
                        @Param("categoryId") Long categoryId);
```

public class BookmarkService {
    public List<BookmarkDto> getBookmarks(Long personId) {
        return bookmarkRepository.findByPersonId(personId)
            .stream().map(this::toDto).collect(Collectors.toList());

        b.setPersonId(personId);
        b.setType(req.getType() != null ? req.getType() : "search");
        b.setName(req.getName());
        b.setRequestUri(req.getRequestUri());

            throw new ResponseStatusException(HttpStatus.FORBIDDEN, "Not bookmark owner");
        }
        bookmarkRepository.delete(b);
    }

}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Untrusted `q` query parameter and bookmark body fields crossing from browser into Spring handlers |
| JWT→service | JWT-extracted `personId` used to scope bookmark ownership — must not be alterable by client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering / Injection | `TicketRepository.searchTickets` — `q` param fed to `plainto_tsquery` | mitigate | `plainto_tsquery('english', :q)` is a named bind parameter in a native @Query — Spring Data JPA uses PreparedStatement parameter binding, never string interpolation. `plainto_tsquery` itself treats the entire value as a phrase (no tsquery operators parsed), making operator-injection impossible. Combined with the 255-char trim in `TicketService`, there is no SQL injection surface here. Guard location: `TicketService.listTickets` (trim) + `TicketRepository` @Query (param binding). |
| T-06-02 | Elevation of privilege / Information disclosure | `BookmarkController.getBookmarks` — returns bookmarks for authenticated user | mitigate | `personId` is read exclusively from the validated JWT via `@AuthenticationPrincipal PersonDetails` — it is never taken from a request parameter or body. `BookmarkRepository.findByPersonId(personId)` ensures the WHERE clause scopes to the JWT owner only. An attacker cannot request another user's bookmarks by changing the URL. Guard location: `BookmarkController.getBookmarks` reads `currentUser.getPersonId()` from the JWT principal. |
| T-06-03 | Elevation of privilege | `BookmarkController.deleteBookmark` — non-owner deletes another user's bookmark | mitigate | `BookmarkService.deleteBookmark` fetches the bookmark, compares `b.getPersonId()` to the JWT `personId`, and throws `ResponseStatusException(403)` if they differ and the user is not ROLE_ADMIN. Guard location: `BookmarkService.deleteBookmark` ownership check before `bookmarkRepository.delete(b)`. |
| T-06-04 | Information disclosure | `searchSnippet` field in TicketListItem — ts_headline may expose ticket description fragments to unauthorized callers | mitigate | `/api/tickets` is already behind JWT authentication (SecurityConfig from Phase 2). Unauthenticated callers receive 401 before reaching the handler; role-unauthorized callers receive 403. No additional change needed; the existing auth layer is the guard. |
| T-06-05 | Denial of service | `q` param containing extremely long strings sent to `plainto_tsquery` causing excess CPU in PostgreSQL | mitigate | `TicketService.listTickets` trims `q` to 255 characters before passing to any repository method. Guard location: `if (trimmedQ.length() > 255) trimmedQ = trimmedQ.substring(0, 255);` in `TicketService`. |
</threat_model>
