**4. Create src/hooks/useDebounce.ts:**
```typescript
import { useState, useEffect } from 'react';

export function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState<T>(value);
  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);
  return debouncedValue;
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Dashboard API responses (stats/chart/clusters/tickets) cross from backend into React state and are rendered to DOM |
| API→map | Geoclusters GeoJSON features from backend are passed to Mapbox/Leaflet for marker rendering |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information disclosure | StatCard count display | mitigate | Stat cards are rendered only inside authenticated AppShell (AuthContext guard in App.tsx route config); unauthenticated requests to /api/dashboard/stats return 401 — handled by React Query's onError and auth redirect |
| T-08-02 | Tampering / XSS | RecentCasesFeed ticket data rendered to DOM | mitigate | React's JSX escapes all text interpolation (ticketId, categoryName, reporterName) automatically — no dangerouslySetInnerHTML used; raw HTML from API is never injected |
| T-08-03 | Tampering / XSS | MapWidget GeoJSON feature properties rendered in Popover | mitigate | Popover content uses JSX text nodes for ticket title/address — no innerHTML used; Mapbox popup content built via DOM API (setDOMContent, not setHTML with raw strings) |
| T-08-04 | Information disclosure | VITE_MAPBOX_TOKEN in client bundle | accept | MAPBOX_ACCESS_TOKEN is a public-facing browser token scoped to URLs (Mapbox token restrictions must be configured in Mapbox dashboard by the operator); it is not a server-side secret — this risk is accepted per Mapbox's own guidance |
| T-08-05 | Elevation of privilege | QuickLinks "Assigned to Me" URL parameter | mitigate | assignedPerson_id in URL is a filter hint; the backend /api/tickets endpoint enforces JWT-based authorization — a crafted URL with another user's ID only shows tickets the backend is already authorized to return |
</threat_model>
