Replace the entire file with:

```bash
#!/usr/bin/env bash
# Dev server for uReport backend — Spring Boot only (no Docker in this sandbox)
# Sandbox facts: Java 21 installed, Maven not pre-installed, runs as root (no sudo needed)

set -euo pipefail

# === Bash version guard ===
if (( BASH_VERSINFO[0] < 4 )) || { (( BASH_VERSINFO[0] == 4 )) && (( BASH_VERSINFO[1] < 3 )); }; then
  echo "[pivota] bash 4.3+ required; found ${BASH_VERSION}" >&2
  exit 127
fi

# === Tee all output to log file ===
mkdir -p /tmp
exec > >(tee -a /tmp/pivota-dev.log) 2>&1
echo "[pivota] $(date -Iseconds) start-dev.sh begin"

# === Detect / set JAVA_HOME ===
# Java 21 is already installed. Resolve JAVA_HOME from the java binary on PATH.
if command -v java >/dev/null 2>&1; then
  JAVA_HOME=$(readlink -f "$(command -v java)" 2>/dev/null | sed -E "s%/bin/java$%%")
  export JAVA_HOME
  export PATH="${JAVA_HOME}/bin:${PATH}"
  echo "[pivota] JAVA_HOME=${JAVA_HOME}"
else
  echo "[pivota] ERROR: java not found; install openjdk-21-jdk-headless and retry" >&2
  exit 1
fi

# === Install Maven if absent (root has direct apt-get access — no sudo) ===
if ! command -v mvn >/dev/null 2>&1; then
  echo "[pivota] installing Maven (one-time, ~30s)"
  apt-get update -qq
  DEBIAN_FRONTEND=noninteractive apt-get install -y -qq maven
fi
echo "[pivota] Maven: $(mvn --version 2>/dev/null | head -1)"

# === Seed JWT_SECRET default for dev (required by application.yml) ===
# application-dev.yml supplies a default, but set the env var here as belt-and-braces
export JWT_SECRET="${JWT_SECRET:-dev-secret-32-characters-minimum-for-hs256}"

# === Start Spring Boot backend with dev profile ===
echo "[pivota] starting Spring Boot backend (spring.profiles.active=dev) on port 8080"
cd backend
exec mvn spring-boot:run -Dspring-boot.run.profiles=dev -Dspring-boot.run.jvmArguments="-DJWT_SECRET=${JWT_SECRET}"
# === END PIVOTA PREAMBLE ===
```

Key rules honored:
- No `sudo` — sandbox runs as root, `apt-get` works directly
- `JAVA_HOME` is only set after confirming `java` is on PATH (guards against `set -u` unbound variable)
- `exec mvn` (not `mvn &`) — single-process, foreground, clean signal propagation
- JWT_SECRET env var set before launch so Spring Boot doesn't fail on `${JWT_SECRET}` placeholder

**Step A — Create `backend/src/main/resources/application-dev.yml`:**

This profile overrides `application.yml` when `--spring.profiles.active=dev` is active. It replaces PostgreSQL (not accessible in this sandbox) with H2 in-memory, disables Flyway (no schema migrations needed for dev smoke-testing), and provides a default JWT_SECRET so the app starts without external secrets.

```yaml
# Spring Boot 'dev' profile — for sandbox / local development without PostgreSQL
# Overrides application.yml — H2 in-memory DB, no Flyway, default JWT secret
spring:
  datasource:
    url: jdbc:h2:mem:devdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE;NON_KEYWORDS=VALUE
    username: sa
    password:
    driver-class-name: org.h2.Driver
  jpa:
    hibernate:
      ddl-auto: create-drop
    properties:
      hibernate:
        dialect: org.hibernate.dialect.H2Dialect
  flyway:
    enabled: false
  h2:
    console:
      enabled: false

management:
  health:
    db:
      enabled: false
    ldap:
      enabled: false
  endpoints:
    web:
      exposure:
        include: health,info
  endpoint:
    health:
      show-details: never

# JWT — default dev secret (32+ chars for HS256)
jwt:
  secret: ${JWT_SECRET:dev-secret-32-characters-minimum-for-hs256}
  expiry-seconds: 28800
  issuer: ureport

# LDAP + CAS disabled — not available in dev sandbox
ldap:
  enabled: false

cas:
  enabled: false
```

Note: `NON_KEYWORDS=VALUE` in the H2 URL avoids a reserved-word clash with any `value` column names in the schema.

**Step B — Fix H2 scope in `backend/pom.xml`:**

H2 is currently declared with `<scope>test</scope>`. That makes it unavailable at runtime (even with `mvn spring-boot:run`), so the dev profile will fail to find the H2 driver. Change it to `<scope>runtime</scope>` so H2 is available for both dev and test:

In `pom.xml`, find:
```xml
        <!-- H2 for in-memory testing -->
        <dependency>
            <groupId>com.h2database</groupId>
            <artifactId>h2</artifactId>
            <scope>test</scope>
        </dependency>
```

Replace `<scope>test</scope>` with `<scope>runtime</scope>`.

`runtime` scope is correct here — H2 is a JDBC driver; it's never referenced directly in application code (only via JDBC URL), so compile scope is unnecessary. Tests continue to work because `runtime` is available on the test classpath.

command -v mvn >/dev/null 2>&1 || (apt-get update -qq && DEBIAN_FRONTEND=noninteractive apt-get install -y -qq maven)
