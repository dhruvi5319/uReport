```yaml
version: "3.9"

services:

  # --- PostgreSQL 16 ---
  db:
    image: postgres:16-alpine
    container_name: ureport-db
    restart: unless-stopped
    environment:
      POSTGRES_DB: ureport
      POSTGRES_USER: ureport
      POSTGRES_PASSWORD: ureport
    volumes:
      - ureport_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ureport -d ureport"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s

  # --- Spring Boot API ---
  api:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: ureport-api
    restart: unless-stopped
    environment:
      SPRING_PROFILES_ACTIVE: docker
      POSTGRES_USER: ureport
      POSTGRES_PASSWORD: ureport
    ports:
      - "8080:8080"
    depends_on:
      db:
        condition: service_healthy
    healthcheck:
      test: ["CMD-SHELL", "wget -qO- http://localhost:8080/actuator/health || exit 1"]
      interval: 15s
      timeout: 10s
```

**`frontend/nginx.conf`** — Nginx reverse proxy config:

```nginx
server {
    listen 80;
    server_name _;

    # Proxy API calls to Spring Boot
    location /api/ {
        proxy_pass         http://api:8080/api/;
        proxy_http_version 1.1;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_connect_timeout 10s;
        proxy_read_timeout    60s;
    }

    # Proxy actuator health check
    location /actuator/ {
        proxy_pass         http://api:8080/actuator/;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;

        index index.html;
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| {env→container} | Database credentials flow from `docker-compose.yml` environment variables into Spring Boot and PostgreSQL containers |
| {nginx→api} | Nginx proxy_pass forwards client HTTP requests to the Spring Boot api container on the internal Docker network |
| {client→nginx} | External HTTP requests on port 80 cross into the Nginx reverse proxy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information disclosure | docker-compose.yml hardcoded credentials (POSTGRES_USER/PASSWORD) | accept | Development-only credentials in compose file are intentional for local dev bootstrapping. Production deployment must use Docker secrets or an `.env` file excluded from git. `.gitignore` must include `.env`. Owner: Phase 9 deployment hardening. |
| T-03-02 | Spoofing | nginx proxy_pass — Host header forwarding | mitigate | nginx.conf sets `proxy_set_header Host $host` and `X-Real-IP $remote_addr` — preserves original client identity. Spring Boot should validate the Host header in Phase 2 CORS config. Location: `frontend/nginx.conf` proxy_pass block. |
| T-03-03 | Denial of service | actuator/health endpoint exposed on port 8080 publicly | mitigate | Only `health` and `info` are exposed via `management.endpoints.web.exposure.include`. Sensitive actuator endpoints (env, beans, heapdump) are NOT exposed. Location: `backend/src/main/resources/application.yml` management.endpoints block. |
| T-03-04 | Elevation of privilege | PostgreSQL port 5432 exposed on host via `ports: - "5432:5432"` | accept | Port 5432 exposed for local dev convenience. Production deployment should remove this binding. Owner: Phase 9 deployment hardening. |
| T-03-05 | Tampering | Docker image supply chain — unpinned `nginx:alpine` and `postgres:16-alpine` tags | accept | Using floating `alpine` tags for dev speed. Production deployment in Phase 9 should pin to digest hashes. Owner: Phase 9. |
</threat_model>
