**SubstatusService.java** (package `com.ureport.admin.service`):

Seeded ids: 1 (Resolved), 2 (Duplicate), 3 (Bogus) — seeded by Flyway V1 migration. These cannot be deleted.

```java
private static final Set<Long> SEEDED_SUBSTATUS_IDS = Set.of(1L, 2L, 3L);

listSubstatuses(): substatusRepository.findAllByOrderByStatusAscNameAsc() → map to SubstatusDto

createSubstatus(CreateSubstatusRequest req):
  - validate status is "open" or "closed" → throw 400 if not
  - if req.isDefault: clear others with same status via clearDefaultForStatus(req.status)
  - create Substatus; substatusRepository.save(s); return SubstatusDto

updateSubstatus(Long id, CreateSubstatusRequest req):
  - load or 404
  - validate status value
  - if req.isDefault: clearDefaultForStatus(req.status) — then set this one's isDefault=true
  - if !req.isDefault: just update fields (don't touch other rows)
  - save; return SubstatusDto

deleteSubstatus(Long id):
  - if SEEDED_SUBSTATUS_IDS.contains(id): throw BusinessException("SEEDED_RECORD_PROTECTED","System substatus cannot be deleted",HttpStatus.FORBIDDEN)
  - load or 404
  - substatusRepository.delete(s)

private void clearDefaultForStatus(String status):
  substatusRepository.findByStatusAndIsDefaultTrue(status)
    .forEach(s -> { s.setIsDefault(false); substatusRepository.save(s); });
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-18 | Elevation of privilege | `POST /api/substatuses`, `PUT /api/substatuses/{id}`, `DELETE /api/substatuses/{id}` and all write endpoints for issue-types, contact-methods, actions | mitigate | `@PreAuthorize("hasRole('ADMIN')")` on all 16 write methods across the 4 controllers; Spring Security method-level enforcement runs before handler body; GET list endpoints require JWT (any authenticated role) |
| T-05-19 | Tampering | `PUT /api/substatuses/{id}` — user sets `isDefault=true` on multiple substatuses for the same status value via concurrent requests | mitigate | `SubstatusService.clearDefaultForStatus(status)` runs within `@Transactional` scope — clears all `isDefault=true` for the status before setting the new one; database-level: a single UPDATE+save sequence; concurrent race is bounded by Postgres row-level locking during the select-for-update in Hibernate |
