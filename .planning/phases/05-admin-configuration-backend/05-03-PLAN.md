**Permission level validation utility** (inline in CategoryService):
Permission level permissiveness order: `anonymous` > `public` > `staff`.
A number: anonymous=0, public=1, staff=2. postingPermissionLevel must have permissiveness ≥ displayPermissionLevel:
```java
private static final Map<String,Integer> PERM_ORDER = Map.of("anonymous",0,"public",1,"staff",2);
private void validatePermissionLevels(String display, String posting) {
  if (PERM_ORDER.get(posting) > PERM_ORDER.get(display)) {
    throw new BusinessException("PERMISSION_LEVEL_INVALID",
      "postingPermissionLevel cannot be more restrictive than displayPermissionLevel", HttpStatus.BAD_REQUEST);
  }
}
```
Example valid: display=public, posting=public ✓ (same). Invalid: display=public, posting=staff ✗ (staff=2 > public=1).

**CategoryService.java** (package `com.ureport.admin.service`):

`listCategories(Long groupId, Long departmentId, Boolean active)`:
- categoryRepository.findFiltered(groupId, departmentId, active)
- Map to List<CategoryListItemDto>

`createCategory(CreateCategoryRequest req)`:
- Validate departmentId not null → departmentRepository.findById(req.departmentId) → 404 if absent
- Validate permission levels via validatePermissionLevels(req.displayPermissionLevel, req.postingPermissionLevel)
- Validate autoClose: if req.autoCloseIsActive && req.autoCloseSubstatusId == null → throw BusinessException("AUTOCLOSE_SUBSTATUS_REQUIRED","autoCloseSubstatusId is required when autoCloseIsActive is true",400)
- Load optional defaultPerson, categoryGroup (404 if provided but not found)
- Create Category, set all scalar fields
- Reconcile actionResponses: for each dto in req.actionResponses, create CategoryActionResponse(categoryId=null — filled by cascade, actionId=dto.actionId, template=dto.template, replyEmail=dto.replyEmail); add to category.getCategoryActionResponses()
- categoryRepository.save(category)
- Return CategoryDetailDto mapped from saved category

`getCategory(Long id)`:
- Load category or 404; return CategoryDetailDto

`updateCategory(Long id, UpdateCategoryRequest req)`:
- Load category or 404
- Validate departmentId, permission levels, autoClose (same as create)
- Update all scalar fields
- **actionResponses reconciliation:** Compare req.actionResponses by id. Items with null/absent id → new CategoryActionResponse added. Items with existing id → update matching entry in category.getCategoryActionResponses(). Items in existing collection not in req → removed (orphanRemoval deletes). 
- categoryRepository.save(category)
- Return CategoryDetailDto

`deleteCategory(Long id)`:
- Load category or 404
- if ticketRepository.existsByCategoryId(id) → throw BusinessException("CATEGORY_IN_USE","Category is referenced by tickets and cannot be deleted",HttpStatus.CONFLICT)
- categoryRepository.delete(category)

`getActionResponse(Long categoryId, Long actionId)`:
- Load category or 404
- Find in category.getCategoryActionResponses() where actionId matches
- If found → return CategoryActionResponseDto with category-specific template
- If not found → actionsRepository.findById(actionId) → return CategoryActionResponseDto with action.template and action.replyEmail as fallback
- If action not found either → 404

@RestController @RequestMapping("/api/categories")
public class CategoryController {

  // GET /api/categories — JWT required
  @GetMapping
  public ResponseEntity<List<CategoryListItemDto>> listCategories(
    @RequestParam(name="group_id",required=false) Long groupId,
    @RequestParam(name="department_id",required=false) Long departmentId,
    @RequestParam(required=false) Boolean active) {...}

  // GET /api/categories/public — NO auth required
  @GetMapping("/public")
  public ResponseEntity<List<CategoryListItemDto>> getPublicCategories() {
    List<Category> cats = categoryRepository.findByPostingPermissionLevelInAndActiveTrue(
      List.of("public","anonymous"));
    return ResponseEntity.ok(cats.stream().map(this::toListItem).collect(toList()));
  }

  // POST /api/categories — admin only
  @PostMapping
  @PreAuthorize("hasRole('ADMIN')")
  public ResponseEntity<CategoryDetailDto> createCategory(
    @Valid @RequestBody CreateCategoryRequest req) {
    return ResponseEntity.status(201).body(categoryService.createCategory(req));
  }

  // GET /api/categories/{id} — JWT required
  @GetMapping("/{id}")
  public ResponseEntity<CategoryDetailDto> getCategory(@PathVariable Long id) {...}

  // PUT /api/categories/{id} — admin only
  @PutMapping("/{id}")
  @PreAuthorize("hasRole('ADMIN')")
  public ResponseEntity<CategoryDetailDto> updateCategory(
    @PathVariable Long id, @Valid @RequestBody UpdateCategoryRequest req) {...}

  // DELETE /api/categories/{id} — admin only
  @DeleteMapping("/{id}")
  @PreAuthorize("hasRole('ADMIN')")
  public ResponseEntity<Void> deleteCategory(@PathVariable Long id) {
    categoryService.deleteCategory(id); return ResponseEntity.noContent().build();
  }

  // GET /api/categories/{id}/action-responses/{actionId} — JWT required

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Untrusted request body (category fields, actionResponses array, permission level strings) crossing into Spring handler |
| anonymous→API | GET /api/categories/public is fully unauthenticated — untrusted request with no JWT |
| JWT→SecurityContext | JWT cookie decoded into PersonDetails for write endpoint authorization |
| API→DB | CategoryRepository JPQL with user-supplied groupId, departmentId, active filter params |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-12 | Elevation of privilege | `POST /api/categories`, `PUT /api/categories/{id}`, `DELETE /api/categories/{id}` | mitigate | `@PreAuthorize("hasRole('ADMIN')")` on all write methods; Spring Security method-level enforcement runs before handler body; GET endpoints require JWT (except /public) |
| T-05-13 | Information disclosure | `GET /api/categories/public` — unauthenticated endpoint | mitigate | Only categories with `postingPermissionLevel IN ('public','anonymous') AND active=true` are returned; internal/staff-only categories (displayPermissionLevel='staff') are never exposed; SecurityConfig `permitAll()` scoped to this single path only |
| T-05-14 | Tampering | `PUT /api/categories/{id}` — actionResponses array with fabricated `id` values pointing to other categories' response rows | mitigate | `CategoryService.updateCategory` reconciliation matches actionResponse items against `category.getCategoryActionResponses()` — only IDs owned by this category's collection are updated; cross-category edit is structurally impossible through the collection scope |
| T-05-15 | Tampering | `POST/PUT /api/categories` — `postingPermissionLevel` can be set to "anonymous" exposing category to public without staff knowing | mitigate | `CategoryService.validatePermissionLevels()` enforces that posting is not more restrictive than display; admin role required for any write; display/posting values must be one of {"staff","public","anonymous"} — invalid enum values throw 400 INVALID_INPUT via GlobalExceptionHandler |
| T-05-16 | Tampering | `GET /api/categories/{id}/action-responses/{actionId}` — returns potentially sensitive response templates | mitigate | JWT required for this endpoint (not in permitAll list); only authenticated staff/admin can retrieve response templates |
| T-05-17 | Denial of service | `GET /api/categories/public` — public endpoint could be hit without rate limiting | accept | Phase 5 scope is admin configuration APIs; rate limiting is an infrastructure concern (ingress/nginx layer). The /public endpoint only reads active categories — no write side effect. Residual risk owned by infrastructure/ops. |
