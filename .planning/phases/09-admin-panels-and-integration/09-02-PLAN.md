**handleLdapSubmit:**
```typescript
async function handleLdapSubmit(e: React.FormEvent<HTMLFormElement>) {
  e.preventDefault();
  setIsLoading(true);
  setErrorMessage(null);
  const data = new FormData(e.currentTarget);
  try {
    const res = await fetch('/api/auth/ldap', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        username: data.get('username'),
        password: data.get('password'),
      }),
      credentials: 'include', // send/receive cookies
    });
    if (res.ok) {
      const returnTo = new URLSearchParams(window.location.search).get('returnTo');
      navigate(returnTo || '/dashboard');
    } else {
      const json = await res.json().catch(() => ({}));
      setErrorMessage(json.error || 'Invalid credentials. Please try again.');
    }
  } catch {
    setErrorMessage('Network error. Please check your connection.');
  } finally {
    setIsLoading(false);
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | LDAP credentials (username/password) from LoginPage form crossing into POST /api/auth/ldap |
| client→API | Bookmark name and requestUri from CaseListPage crossing into POST /api/bookmarks |
| db→render | Bookmark names and requestUri values stored in DB rendered into dropdown; ticket data from search rendered in Command palette |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Information disclosure | LoginPage credential submission | mitigate | POST /api/auth/ldap uses `credentials: 'include'` for cookie-based auth. Password is never stored in React state beyond the form value; no logging of the request body. HTTPS enforced at nginx/ingress level (transfer: infra team). |
| T-09-07 | Spoofing | LoginPage open redirect via ?returnTo | mitigate | `returnTo` is passed to `navigate()` (react-router), which only performs client-side navigation; absolute URLs are ignored by react-router navigate (it only handles relative paths). This prevents open redirect to external domains. Implementation in `LoginPage.tsx::handleLdapSubmit` — verify `navigate(returnTo || '/dashboard')` uses react-router navigate, not `window.location.href`. |
| T-09-08 | Tampering | Bookmark requestUri restored as URL searchParams | mitigate | `bookmark.requestUri` is parsed via `new URLSearchParams(...)` and applied via `setSearchParams()` (react-router). Only valid query parameters are restored; no HTML injection possible since values go to URL bar, not innerHTML. Implementation in `CaseListPage.tsx` bookmark recall handler. |
| T-09-09 | Information disclosure | Command palette ticket search results | transfer | GET /api/tickets?q= requires a valid JWT cookie (Phase 2 Spring Security); unauthenticated users get 401 before any ticket data is returned. Frontend AuthContext already redirects to /login if user is null before Navbar renders. |
| T-09-10 | Denial of service | Command palette fires search on every keystroke | mitigate | 300ms debounce in Navbar.tsx before firing GET /api/tickets?q=; query only enabled when commandQuery.length >= 2. Reduces API calls by ~90% under fast typing. Implementation in `Navbar.tsx::commandQuery` debounce state. |
</threat_model>
