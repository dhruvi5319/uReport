<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Untrusted form data (names, URLs, templates, email addresses) from admin Sheet forms crossing into PUT/POST/DELETE API calls |
| db→render | People names, department names, action templates stored in DB rendered back into admin table cells and Sheet forms |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Elevation of privilege | AdminGuard in App.tsx | mitigate | `AdminGuard` component in `frontend/src/App.tsx` checks `useAuth().user?.role === 'ROLE_ADMIN'`; non-admin gets `<Navigate to="/dashboard">`. Backend API also enforces ROLE_ADMIN via Spring Security (Phase 2, 05); frontend guard is defense-in-depth only. |
| T-09-02 | Information disclosure | ClientsPage API key display | mitigate | `apiKey` stored in ephemeral React component state only; cleared when Sheet closes; never written to localStorage or logged; backend returns masked `••••••••` on subsequent GET — enforced in `ClientService` (Phase 4 04-04). |
| T-09-03 | Tampering | Action template field (Textarea) | mitigate | Template content is stored as plain text and rendered via `{template}` React text node (not `dangerouslySetInnerHTML`), preventing XSS injection when previewed in the table or Sheet. |
| T-09-04 | Information disclosure | People sub-resource data (emails/phones/addresses) | transfer | Personal contact data is returned only when user holds a valid JWT cookie with ROLE_ADMIN or ROLE_STAFF; enforced by Spring Security at the API layer (Phase 2). Frontend admin panel only fetches when authenticated per AuthContext. |
| T-09-05 | Denial of service | Inline fetch on ComboBox search (DepartmentsPage defaultPerson) | accept | Combobox fires GET /api/people?q={term} on keystroke; no server-side rate limiting implemented. Acceptable: admin-only endpoint; low traffic expected. Risk owned by operations team. |
</threat_model>
