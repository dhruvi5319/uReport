**MIME magic bytes constants (exact byte sequences to check):**
```java
private static final byte[] JPEG_MAGIC = new byte[]{(byte)0xFF, (byte)0xD8, (byte)0xFF};
private static final byte[] PNG_MAGIC  = new byte[]{(byte)0x89, 0x50, 0x4E, 0x47};
private static final byte[] GIF_MAGIC  = new byte[]{0x47, 0x49, 0x46};

private String detectMimeType(byte[] header) {
    if (startsWith(header, JPEG_MAGIC)) return "image/jpeg";
    if (startsWith(header, PNG_MAGIC))  return "image/png";
    if (startsWith(header, GIF_MAGIC))  return "image/gif";
    return null; // unsupported
}
private boolean startsWith(byte[] data, byte[] prefix) {
    if (data.length < prefix.length) return false;
    for (int i = 0; i < prefix.length; i++) if (data[i] != prefix[i]) return false;
    return true;
}
```

**MediaService.upload(Long ticketId, List&lt;MultipartFile&gt; files, PersonDetails currentUser):**
1. Load ticket by ticketId → throw 404 TICKET_NOT_FOUND if absent
2. Validate count: files.size() > 10 → throw 400 TOO_MANY_FILES ("Maximum 10 files per request")
3. For each file:
   a. Read first 8 bytes: `byte[] header = new byte[8]; file.getInputStream().read(header)`
   b. String mimeType = detectMimeType(header); if null → throw 400 INVALID_MIME_TYPE
   c. if file.getSize() > 10_485_760 → throw 400 FILE_TOO_LARGE ("Maximum file size is 10 MB")
   d. String ext = FilenameUtils.getExtension(file.getOriginalFilename()); // use Apache Commons IO (already in classpath via Spring) or simple lastIndexOf(".")
   e. String internalFilename = UUID.randomUUID().toString() + (ext.isEmpty() ? "" : "." + ext)
   f. Path dir = Paths.get(mediaRoot, String.valueOf(ticketId)); Files.createDirectories(dir)
   g. Path filePath = dir.resolve(internalFilename); Files.copy(file.getInputStream(), filePath, REPLACE_EXISTING)
   h. Generate thumbnail: `Thumbnails.of(filePath.toFile()).size(150, 150).toFile(dir.resolve("thumb_" + internalFilename).toFile())`; wrap in try/catch: thumbnail failure logs warning but does not block
   i. Build Media entity: ticket_id, original_filename=file.getOriginalFilename(), internal_filename=internalFilename, mime_type=mimeType, size_bytes=file.getSize(), uploaded_date=NOW()
   j. mediaRepository.save(media)
4. Create ONE TicketHistory entry with action "upload_media" for the batch (not one per file)
5. Return List&lt;MediaDto&gt; of all saved media

**MediaService.serveMedia(Long mediaId, boolean thumbnail):**
1. Load Media by mediaId → throw 404 MEDIA_NOT_FOUND if absent
2. Build path: thumbnail ? `{mediaRoot}/{ticketId}/thumb_{internalFilename}` : `{mediaRoot}/{ticketId}/{internalFilename}`
3. If file does not exist on disk → throw 404 MEDIA_FILE_NOT_FOUND
4. Return ResponseEntity with InputStreamResource, Content-Type = media.mime_type (or "image/jpeg" for thumbnail), Content-Length, Content-Disposition: inline

**MediaService.deleteMedia(Long mediaId, PersonDetails currentUser):**
1. Load Media by mediaId → throw 404 MEDIA_NOT_FOUND
2. Delete disk file at `{mediaRoot}/{ticketId}/{internalFilename}` (Files.deleteIfExists)
3. Delete thumbnail at `{mediaRoot}/{ticketId}/thumb_{internalFilename}` (Files.deleteIfExists)
4. mediaRepository.delete(media)

**Inject:** @Value("${media.root:/var/ureport/media}") String mediaRoot; MediaRepository; TicketRepository; TicketHistoryRepository; ActionRepository (for "upload_media" lookup).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Untrusted multipart file bytes entering the upload handler |
| API→filesystem | User-controlled filename and ticket_id path segments entering file storage path |
| client→API | Untrusted api_key UUID in clientRepository — must not be guessable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-18 | Tampering | `MediaService.upload` — MIME type determined by Content-Type header (spoofable) | mitigate | MIME detection uses magic bytes exclusively (`detectMimeType(byte[] header)` in `MediaService`); Content-Type header is ignored for validation |
| T-04-19 | Tampering / Elevation | `MediaService.upload` — ticket_id path segment used in file path construction | mitigate | `ticketId` is a `Long` from the database (not user-supplied string); path built as `Paths.get(mediaRoot, String.valueOf(ticketId))` with no user string interpolation; parent dir canonicalized via `Files.createDirectories` |
| T-04-20 | Information disclosure | `GET /api/media/{mediaId}` — serves raw file; no auth check | mitigate | Consistent with Phase 3 Open311 design: media served publicly for tickets on public categories. For staff-category tickets: `MediaServeController.serveMedia` checks ticket category's `posting_permission` level; if staff-only, JWT required |
| T-04-21 | Denial of service | Large file upload consuming disk space (>10 MB per file, 10 files max = 100 MB per request) | mitigate | `file.getSize() > 10_485_760` guard in `MediaService.upload` rejects before writing to disk; `files.size() > 10` guard limits per-request batch |
| T-04-22 | Information disclosure | `POST /api/clients` returns api_key in plaintext; must not be logged | mitigate | `ClientDetailDto.apiKey` excluded from application log output via `@JsonIgnore` on any logging serialization; api_key stored as-is in DB (Open311 spec requirement — clients send it as HTTP param); only returned on create |
| T-04-23 | Path traversal | `internalFilename` derived from original filename extension — attacker uploads file named `../../etc/passwd` | mitigate | `internalFilename` is UUID-only: `UUID.randomUUID().toString() + "." + ext`; extension extracted as characters after last "." with no path separators (strip all "/" and "\" from ext before use); original filename stored in `original_filename` column but never used as disk path |
| T-04-24 | Elevation of privilege | `/api/clients` endpoints accessible to STAFF role | mitigate | `@PreAuthorize("hasRole('ADMIN')")` on every method in `ClientController`; STAFF role returns 403 confirmed by ClientCrudIT test case 6 |
</threat_model>
