**BulkTicketRequest DTO (exact spec):**
```typescript
interface BulkTicketRequest {
  ticketIds: number[];       // required; min 1
  action: "assign" | "close" | "changeStatus";
  assignedPersonId?: number; // for action=assign
  substatusId?: number;      // for action=close
  status?: string;           // for action=changeStatus ("open" | "closed")
}
```

**BulkOperationResult DTO:**
```java
public class BulkOperationResult {
  private int successCount;
  private int failureCount;
  private List<TicketOperationResult> perTicketResults;
}
public class TicketOperationResult {
  private Long ticketId;
  private boolean success;
  private String error; // null on success
}
```

**TicketBulkService implementation rules:**

`bulkAssign(List<Long> ticketIds, Long personId, PersonDetails currentUser)`:
1. Validate personId: load Person, throw 400 PERSON_NOT_FOUND if absent
2. For each ticketId in ticketIds — wrap in try/catch (per-ticket isolation):
   a. Load ticket; if not found: add TicketOperationResult(ticketId, false, "TICKET_NOT_FOUND")
   b. Update ticket.assigned_person_id=personId, last_modified=NOW()
   c. Create TicketHistory with action "assignment", entered_by_person_id=currentUser.id
   d. ticketRepository.save(ticket), ticketHistoryRepository.save(history)
   e. Add TicketOperationResult(ticketId, true, null)
3. Return BulkOperationResult(successCount, failureCount, perTicketResults)

`bulkClose(List<Long> ticketIds, Long substatusId, String notes, PersonDetails currentUser)`:
1. Validate substatusId: load Substatus, throw 400 SUBSTATUS_NOT_FOUND if absent
2. For each ticketId — wrap in try/catch:
   a. Load ticket; if not found or already closed: record failure
   b. Set ticket: status=closed, substatus_id=substatusId, closed_date=NOW(), last_modified=NOW()
   c. Create TicketHistory with action "closed", notes=notes
   d. Persist both
   e. Add success result
3. Return BulkOperationResult

`bulkChangeStatus(List<Long> ticketIds, String status, PersonDetails currentUser)`:
1. Validate status in ["open","closed"]; throw 400 INVALID_STATUS if not
2. For each ticketId — wrap in try/catch:
   a. Load ticket; if not found: record failure
   b. Set ticket.status=status, last_modified=NOW()
   c. Create TicketHistory with action "update"
   d. Persist both
   e. Add success result
3. Return BulkOperationResult

**IMPORTANT:** Do NOT use @Transactional at the service class level for bulk methods. Use explicit per-ticket save calls inside try/catch. A failure on one ticket must not roll back the others. Annotate individual repository.save() calls within a local try block.

**SlaService implementation rules:**

`computeSlaDueDate(Ticket ticket, Category category)`:
- If category.sla_days is null or 0 → return null (no SLA)
- Return ticket.entered_date.toInstant().plus(category.sla_days, ChronoUnit.DAYS) as OffsetDateTime

`isOverdue(Ticket ticket, Category category)`:
- If ticket.status.equals("closed") → return false
- Instant slaDue = computeSlaDueDate(ticket, category)
- If slaDue == null → return false
- Return Instant.now().isAfter(slaDue.toInstant())

`triggerAutoClose(Ticket ticket, Category category, PersonDetails systemUser)`:
- If category.auto_close_is_active == false → return immediately (no-op)
- If NOT isOverdue(ticket, category) → return
- If ticket.status.equals("closed") → return (already closed)
- Load Substatus by category.auto_close_substatus_id; if null → log warning and return
- Set ticket: status=closed, substatus_id=autoCloseSubstatusId, closed_date=NOW(), last_modified=NOW()
- Create TicketHistory with action "closed", notes="Auto-closed by SLA policy"
- ticketRepository.save(ticket); ticketHistoryRepository.save(history)

**SlaService integration points:**
- Called by TicketService.createTicket (after persist, invoke triggerAutoClose)
- Called by TicketService.getTicket to compute isOverdue / slaDueDate
- Can be called by a future @Scheduled job without modification

**TicketBulkController:**
```
POST /api/tickets/bulk → @PreAuthorize("hasAnyRole('STAFF','ADMIN')")
```
- Accept BulkTicketRequest body
- Validate: ticketIds not empty (throw 400 TICKETIDS_REQUIRED); action not null
- Route to ticketBulkService based on action:
  - "assign" → bulkAssign(req.ticketIds, req.assignedPersonId, currentUser); validate assignedPersonId not null
  - "close" → bulkClose(req.ticketIds, req.substatusId, null, currentUser); validate substatusId not null
  - "changeStatus" → bulkChangeStatus(req.ticketIds, req.status, currentUser); validate status not null
- Return 200 BulkOperationResult
