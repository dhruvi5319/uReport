**NotificationService implementation:**

`getNotificationEmails(Long personId)`:
- Query people_emails table via personEmailsRepository.findByPersonIdAndUsedForNotificationsTrue(personId)
- Return List&lt;String&gt; of email addresses; return empty list if none

`resolveReplyEmail(Long categoryId, Long actionId)`:
- Load CategoryActionResponse by (categoryId, actionId); if present and reply_email not blank → return it
- Else load Action by actionId; if action.reply_email not blank → return it
- Else return system default from @Value("${notification.reply-email:noreply@ureport.local}")

`sendEmail(String to, String subject, String body, String replyTo)`:
- Build SimpleMailMessage; set to, subject, text, replyTo
- Call javaMailSender.send(message)
- Wrap in try/catch(MailException); on exception: log.warn("Email delivery failed to {}: {}", to, e.getMessage())
- NEVER re-throw; NEVER let email failure propagate to caller

`sendTicketNotification(Long ticketId, TicketHistory historyEntry, boolean notifyReporter, boolean notifyAssignee)`:
- Load ticket; resolve reply email for category/action pair
- Build List&lt;String&gt; sentTo
- If notifyReporter: load ticket.reported_by_person_id → getNotificationEmails(personId) → sendEmail each
- If notifyAssignee: load ticket.assigned_person_id → getNotificationEmails(personId) → sendEmail each
- Return sentTo (addresses actually attempted; not filtered by success, since sendEmail is non-fatal)

**Mail subject template:** `"[Case #${ticketId}] ${action.name}: ${ticket.category.name}"`
**Mail body template:** Use historyEntry.notes if present, else action.name

Inject: JavaMailSender (auto-configured by Spring Boot when starter present), @Value("${notification.reply-email:noreply@ureport.local}").

**TicketHistoryService implementation:**

`logAction(Long ticketId, LogActionRequest req, PersonDetails currentUser)`:
1. Load ticket by ticketId → throw 404 TICKET_NOT_FOUND if absent
2. Load Action by req.actionId → throw 400 ACTION_NOT_FOUND if absent
3. Validate "response" rule: if action.name.equals("response") && (req.notes == null || req.notes.isBlank()) → throw 400 NOTES_REQUIRED
4. Department filter (skip for ADMIN role):
   - Get ticket's category → category.department_id
   - Call departmentActionRepository.existsByDepartmentIdAndActionId(dept_id, req.actionId)
   - If false → throw 403 DEPARTMENT_ACTION_FORBIDDEN
5. Build TicketHistory:
   - ticket_id = ticketId
   - action_id = req.actionId
   - entered_by_person_id = currentUser.id
   - action_person_id = req.actionPersonId != null ? req.actionPersonId : currentUser.id
   - entered_date = NOW()
   - action_date = NOW()
   - notes = req.notes
6. List&lt;String&gt; sentTo = notificationService.sendTicketNotification(ticketId, history, notifyReporter, notifyAssignee)
7. Set history.sent_notifications = ObjectMapper.writeValueAsString(sentTo)  // JSON array
8. ticketHistoryRepository.save(history)
9. Return map to TicketHistoryEntryDto

`getTimeline(Long ticketId)`:
- Load all TicketHistory by ticketId ordered by entered_date DESC
- Return List&lt;TicketHistoryEntryDto&gt;

`getResponseTemplate(Long categoryId, Long actionId)`:
- Find CategoryActionResponse by (categoryId, actionId) → return response_text if present
- Else find Action by actionId → return action.template if present
- Else return null

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Untrusted actionId, notes, notifyReporter flags entering POST /api/tickets/{id}/history |
| API→SMTP | Email addresses resolved from DB crossing into JavaMailSender SMTP call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-12 | Elevation of privilege | `TicketHistoryService.logAction` — department_actions filter | mitigate | `departmentActionRepository.existsByDepartmentIdAndActionId` enforced for every STAFF user; ADMIN role explicitly bypasses via `currentUser.getAuthorities().contains("ROLE_ADMIN")` check before query |
| T-04-13 | Tampering | `LogActionRequest.actionPersonId` — attacker sets this to another user's ID to forge attribution | mitigate | `actionPersonId` is validated: if present, load Person by ID; if not found throw 400; logged for audit but does not change actor permissions |
| T-04-14 | Information disclosure | Notification emails — email addresses from people_emails could leak via sentNotifications JSON stored in history | mitigate | Access to `GET /api/tickets/{id}/history` requires valid JWT (`@Authenticated`); sentNotifications field returned only to authenticated users; no public endpoint exposes history |
| T-04-15 | Denial of service | SMTP calls blocking request thread if SMTP server is slow | mitigate | `spring.mail.properties.mail.smtp.timeout=5000` in application.yml caps connection+read time; MailException caught and logged; non-fatal pattern in `NotificationService.sendEmail` |
| T-04-16 | Tampering | `notifyReporter`/`notifyAssignee` flags — attacker triggers spam to reporter by repeated history posts | mitigate | `@PreAuthorize("hasAnyRole('STAFF','ADMIN')")` on POST endpoint; only authenticated staff can trigger notifications; no unauthenticated path to email dispatch |
| T-04-17 | Information disclosure | `GET /api/categories/{id}/action-responses/{actionId}` — returns template text that may contain internal workflow notes | mitigate | JWT required for endpoint (`@Authenticated`); templates are configuration data; no PII embedded (by convention enforced in admin UI Phase 9) |
</threat_model>
